# Plot Color Matching Notes

## Problem

Plots can receive the same default color palette but still assign different colors to the same sample groups. The palette itself is not the root problem. The mismatch happens because group-to-color assignment is handled differently by the plotting backends.

- `plot_pca_2d()` uses `ggplot2::scale_colour_manual(values = color_values)`.
- `plot_pca_3d()` passes `colors = color_values` to `plotly::plot_ly()`.
- When `color_values` is unnamed, ggplot and plotly may assign the same palette to group labels using different category ordering or trace handling.

The expected default behavior is that every group-colored plot assigns the same observed group labels to the same default colors. For example, with groups first observed as `A`, `B`, and `C`, all outputs should consistently map:

```text
A -> #5954d6
B -> #e1562c
C -> #b80058
```

This should apply to:

- standalone 2D PCA capsule output
- standalone 3D PCA capsule output
- histogram output colored by group or sample
- heatmap annotation colors
- PCA output generated by the filter capsule/package flow
- PCA output generated by the normalization capsule/package flow

## Package-Level Change Needed

Fix this in the MOSuite package instead of in individual capsule wrappers.

Added one shared helper, `resolve_plot_colors()`, that normalizes plot colors before plotting. The helper:

1. Accept `color_values` and the observed group labels.
2. Preserve existing names when `color_values` is already a named vector for those labels.
3. Treat named vectors whose names do not cover the observed groups as palette-label names for compatibility.
4. When `color_values` is unnamed, assign names deterministically using first-observed group labels, matching `get_colors_vctr()`.
5. Error when there are more observed groups than colors.

Implemented helper shape:

```r
resolve_plot_colors(dat, colname, color_values = NULL, palette_fun = grDevices::palette.colors, ...)
```

Location: [R/colors.R](R/colors.R).

## Files To Inspect Or Change

- [R/plot_pca.R](R/plot_pca.R)
  - Applies shared color matching before `ggplot2::scale_colour_manual()` in `plot_pca_2d()`.
  - Applies shared color matching before `plotly::plot_ly(colors = ...)` in `plot_pca_3d()`.

- [R/filter.R](R/filter.R)
  - Filter-generated PCA passes `colors_for_plots` into `plot_pca()`.
  - Removed `as.vector(colors_for_plots)` because it stripped group names and recreated the mismatch.

- [R/normalize.R](R/normalize.R)
  - Normalization-generated PCA also passes `colors_for_plots` into `plot_pca()`.
  - Existing forwarding preserves named colors; tests cover that path.

- [R/plot_histogram.R](R/plot_histogram.R)
  - Resolves group-colored and sample-colored histograms before `ggplot2::scale_colour_manual()`.

- [R/plot_heatmap.R](R/plot_heatmap.R)
  - Resolves correlation and expression heatmap annotation colors.

- [R/batch-correction.R](R/batch-correction.R)
  - Not part of the immediate capsule request, but it also creates PCA plots with `colors_for_plots`. Check it for the same pattern while fixing all PCA outputs.

## Tests To Add

Added tests in [tests/testthat/test-colors.R](tests/testthat/test-colors.R), [tests/testthat/test-plot_pca.R](tests/testthat/test-plot_pca.R), [tests/testthat/test-plot_histogram.R](tests/testthat/test-plot_histogram.R), [tests/testthat/test-plot_heatmap.R](tests/testthat/test-plot_heatmap.R), [tests/testthat/test-filter.R](tests/testthat/test-filter.R), and [tests/testthat/test-normalize.R](tests/testthat/test-normalize.R).

Coverage:

1. `plot_pca_2d()` with unnamed default colors maps groups `A`, `B`, `C` to the first three default colors.
2. `plot_pca_3d()` with unnamed default colors maps groups `A`, `B`, `C` to the same first three default colors.
3. Named colors are preserved exactly when passed in.
4. Filter-generated PCA output preserves group color names.
5. Normalization-generated PCA output preserves group color names.
6. Histogram group and sample colors use first-observed order.
7. Correlation and expression heatmap annotation colors use first-observed order.
8. Resolver errors on too few colors and treats non-category names as palette labels.

The important assertion is not that the palette strings match. The assertion must check the group-to-color mapping.

## Capsule Follow-Up

A temporary wrapper-level fix was made in the 3D PCA capsule while diagnosing the mismatch. Once the package-level fix lands and the capsule uses the updated MOSuite package, remove that wrapper workaround so color matching is owned by MOSuite itself.
